# gpt-oss-120b EAGLE3 Draft Training on 8×B200

Reproduce the EAGLE3 speculative-decoding draft model for
[`openai/gpt-oss-120b`](https://huggingface.co/openai/gpt-oss-120b),
matching the configuration of NVIDIA's reference checkpoint
[`nvidia/gpt-oss-120b-Eagle3-long-context`](https://huggingface.co/nvidia/gpt-oss-120b-Eagle3-long-context).

Training uses **NVIDIA Model Optimizer (ModelOpt)** with **on-the-fly hidden
state collection** — the 120B target and the 215M draft co-reside on a single
8×B200 node, so no multi-TB hidden-state cache hits disk.

---

## Overview

| Component | Spec |
| --- | --- |
| Target model | `openai/gpt-oss-120b` (36-layer MoE, hidden_size=2880, vocab=201088) |
| Draft model | 1-layer Llama (`LlamaForCausalLMEagle3`), ~215M params, bf16 |
| Aux hidden states | layers **[1, 17, 32]** of target (low / mid / high) |
| RoPE | llama3 scaling, factor=8, original_max=8192, theta=500000 |
| Max position | 131072 |
| Training data | Nemotron Post-Training V2, 500k samples (multilingual ×0.1), responses regenerated by target |
| Sampling | per-sample `reasoning_effort ∈ {low,med,high}`, `temperature ~ U(0,1)` |
| Optimizer | AdamW, lr 3e-4 → 1e-4 (cosine, warmup 3%) |
| Batch | global 128, 3 epochs |
| Hardware | 8 × B200 (180 GB), bf16 |
| Reference τ | MT-Bench 1.95–2.83 @ draft_len=3 |

Source paper: <https://arxiv.org/html/2604.09557v1> (Appendix H).

- 8 × B200 (180 GB) or H100/H200 80 GB (will need TP adjustments)

- HuggingFace token with access to `openai/gpt-oss-120b`

## Quickstart — 5k Dry-Run

End-to-end sanity check; ~3 hours total on 8×B200.

```bash
# 1. Environment + model download (~30 min, mostly download)
bash 00_env/setup.sh
source ~/eagle3-gptoss/.venv/bin/activate     # path created by setup.sh

# 2. Sub-sample 5k prompts from Nemotron V2
python 01_data/prepare.py --n 5000

# 3. Regenerate responses with gpt-oss-120b (vLLM TP=8, ~1–2 h)
python 01_data/regenerate.py

# 4. Train EAGLE3 draft (1 epoch, ~30 min for 5k)
bash 02_train/train.sh

# 5. Evaluate acceptance length on MT-Bench
python 03_eval/eval_acceptance.py \
    --draft ./ckpt/dry-run
```

A τ ≥ 2.0 on the dry-run is enough to validate the pipeline. Scale up to
500k and 3 epochs once green.

## Key Design Notes

### Why these aux layer ids?
`[1, 17, 32]` is shipped by NVIDIA's reference checkpoint. With 36 target
layers this is "early / mid / late" rather than even thirds — early layer 1
captures token-level features, layer 17 mid-stack semantics, layer 32 the
near-final residual. Changing these requires re-collecting hidden states.

### Why llama3 RoPE (not YaRN)?
The reference long-context checkpoint uses `rope_type: llama3` with
`factor=8, original_max_position_embeddings=8192`. This extends effective
context to 65536; combined with `max_position_embeddings=131072` it covers
the deployed range. The Eagle3 paper notes that earlier public checkpoints
degraded at long context due to RoPE misconfiguration — preserve this block
exactly when modifying `configs/draft_config.json`.

### Why on-the-fly hidden states?
Offline cache for 500k × 8k tokens × 3 layers × 2880 dim × bf16 ≈ 25 TB.
With 8×B200 we have plenty of HBM to keep the target resident
(TP=4, ~60 GB / GPU) while the draft replicates across all 8 GPUs (DP=8).
Cost: ~30% slower step than offline, no disk pressure.

### Vocab pruning
Disabled (`draft_vocab_size: null`). Matches reference checkpoint. Pruning
can boost throughput further but requires retraining; leave as future work.

## Evaluation Targets

From the [HF model card](https://huggingface.co/nvidia/gpt-oss-120b-Eagle3-long-context),
MT-Bench acceptance rate at `draft_len=3`: **1.95–2.83 tokens/step** across
categories. The eval script prints τ = 1 + draft_len × accept_rate.

Deployment with TensorRT-LLM:

```bash
trtllm-serve <gpt-oss-120b checkpoint> \
  --host 0.0.0.0 --port 8000 \
  --backend pytorch \
  --max_batch_size 32 --max_num_tokens 8192 --max_seq_len 8192 \
  --tp_size 8 \
  --extra_llm_api_options - <<'YAML'
speculative_config:
  decoding_type: Eagle
  max_draft_len: 3
  speculative_model_dir: ./ckpt/full
YAML
```

## Troubleshooting

| Symptom | Likely cause | Fix |
| --- | --- | --- |
| `ModuleNotFoundError: modelopt.torch.speculative.eagle.train` | ModelOpt version mismatch | `python -m modelopt.torch.speculative.eagle --help` to find correct entry; update `02_train/train.sh` |
| OOM on target during on-the-fly | TP/DP misplanned | Drop `--target_tp/--draft_dp`, let ModelOpt auto-plan, or raise `--target_tp` to 8 |
| τ < 1.5 after full training | Aux layer ids / RoPE drift from target | Verify `configs/draft_config.json` matches the HF reference byte-for-byte |
| Long-context τ collapses | RoPE scaling mismatch | Re-confirm `rope_scaling` block; never swap llama3 ↔ YaRN |
| vLLM regenerate slow | Prefix cache off | Confirm `enable_prefix_caching=True` (default in `01_data/regenerate.py`) |
